## 1. 前提条件

- Ubuntu 24.04 LTS（x86_64）
- root または sudo 権限
- 外部公開する場合: ドメイン名のDNS Aレコードがサーバーの公開IPに向いていること
- HTTPS取得時: ルーターでポート80をサーバーに転送していること

## 2. TimescaleDB + PostgreSQL インストール

```bash
# 必要パッケージのインストール
apt-get update
apt-get install -y gnupg curl ca-certificates

# TimescaleDB リポジトリの登録
curl -fsSL https://packagecloud.io/timescale/timescaledb/gpgkey \
  | gpg --dearmor -o /usr/share/keyrings/timescaledb.gpg

echo 'deb [signed-by=/usr/share/keyrings/timescaledb.gpg] https://packagecloud.io/timescale/timescaledb/ubuntu/ noble main' \
  > /etc/apt/sources.list.d/timescaledb.list

# TimescaleDB + PostgreSQL 16 インストール
apt-get update
apt-get install -y timescaledb-2-postgresql-16 postgresql-client-16
```

## 3. データベースセットアップ

### 3-1. PostgreSQL 設定

```bash
# TimescaleDB をプリロードに追加
sed -i "s/#shared_preload_libraries = ''/shared_preload_libraries = 'timescaledb'/" \
  /etc/postgresql/16/main/postgresql.conf

# pg_hba.conf: postgres ユーザーを一時的に trust 認証に変更
sed -i 's/^local   all             postgres                                peer/local   all             postgres                                trust/' \
  /etc/postgresql/16/main/pg_hba.conf

# PostgreSQL 起動
service postgresql start
service postgresql reload
```

### 3-2. senhub ユーザー・DB 作成

```bash
psql -U postgres -c "CREATE USER senhub WITH PASSWORD 'senhubpass';"
psql -U postgres -c "CREATE DATABASE senhub OWNER senhub;"
psql -U postgres -d senhub -c "CREATE EXTENSION IF NOT EXISTS timescaledb CASCADE;"
```

### 3-3. pg_hba.conf を元に戻す

```bash
# peer 認証に戻す（セキュリティのため必ず実施）
sed -i 's/^local   all             postgres                                trust/local   all             postgres                                peer/' \
  /etc/postgresql/16/main/pg_hba.conf

service postgresql reload
```

## 4. スキーマ適用

```bash
# リポジトリから schema.sql を取得、または scp で転送
PGPASSWORD=senhubpass psql -U senhub -d senhub -h 127.0.0.1 -f server/schema.sql
```

作成されるテーブル・ビュー：

| 名前 | 種別 | 内容 |
|------|------|------|
| `channels` | テーブル | チャンネル認証キー管理 |
| `channel_properties` | テーブル | フィールドのメタ情報（name/unit/type） |
| `sensor_data` | Hypertable | 連続センサーデータ |
| `events` | Hypertable | ON/OFF イベントログ |
| `sensor_data_1min` | 集約ビュー | 1分平均（Continuous Aggregate） |
| `sensor_data_1hour` | 集約ビュー | 1時間平均（Continuous Aggregate） |

## 5. サーバーファイルのデプロイ

```bash
# デプロイ先ディレクトリを作成
mkdir -p /opt/senhub/server

# ファイルをコピー（リポジトリまたは scp）
cp server/main.py     /opt/senhub/server/
cp server/db.py       /opt/senhub/server/
cp server/config.py   /opt/senhub/server/
cp server/channels.yaml /opt/senhub/server/   # ← 秘密鍵を含むため取り扱い注意
```

> **注意:** `channels.yaml` はWRITE/READキーを含む秘密ファイルです。
> パーミッションを制限してください:
> ```bash
> chmod 600 /opt/senhub/server/channels.yaml
> ```

## 6. Python パッケージのインストール

```bash
# pip3 が未インストールの場合
apt-get install -y python3-pip
```

|-----------|------|

## 7. 設定ファイルの作成

### HTTP モード（社内LAN・開発環境）

```bash
cat > /opt/senhub/server/.env << 'EOF'
SENHUB_USE_TLS=false
SENHUB_PORT=8000
SENHUB_DOMAIN=192.168.0.92
SENHUB_DB_URL=postgresql://senhub:senhubpass@127.0.0.1:5432/senhub
EOF
```

### HTTPS モード（本番・証明書取得後）

```bash
cat > /opt/senhub/server/.env << 'EOF'
SENHUB_USE_TLS=true
SENHUB_PORT=443
SENHUB_DOMAIN=senhub.hide23.link
SENHUB_DB_URL=postgresql://senhub:senhubpass@127.0.0.1:5432/senhub
EOF
```

## 8. systemd サービス登録（常時起動）

```bash
cat > /etc/systemd/system/senhub.service << 'EOF'
[Unit]
Description=Senhub IoT Server
After=network.target postgresql.service
Requires=postgresql.service

[Service]
Type=simple
WorkingDirectory=/opt/senhub/server
ExecStart=/usr/bin/python3 /opt/senhub/server/main.py
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal
SyslogIdentifier=senhub

[Install]
WantedBy=multi-user.target
EOF

# サービスを有効化・起動
systemctl daemon-reload
systemctl enable senhub
systemctl start senhub

# 起動確認
systemctl status senhub
```

## 9. HTTPS 対応（Let's Encrypt）

### 前提
- ドメイン（例: `senhub.hide23.link`）のDNS AレコードがサーバーのグローバルIPに向いている
- ルーターで **ポート80** をサーバーに転送している

### 9-1. certbot インストール

```bash
apt-get install -y certbot
```

### 9-2. サービスを一時停止して証明書取得

```bash
# senhub を停止（ポート80を certbot が使うため）
systemctl stop senhub

# ポート80を開放
ufw allow 80

# 証明書取得
certbot certonly --standalone \
  --non-interactive \
  --agree-tos \
  --email your-email@example.com \
  -d senhub.hide23.link
```

取得した証明書のパス：
```
/etc/letsencrypt/live/senhub.hide23.link/fullchain.pem
/etc/letsencrypt/live/senhub.hide23.link/privkey.pem
```

> 証明書は **90日ごとに自動更新**されます（certbot.timer が自動設定済み）。

### 9-3. .env を HTTPS 設定に更新してサービス再起動

```bash
cat > /opt/senhub/server/.env << 'EOF'
SENHUB_USE_TLS=true
SENHUB_PORT=443
SENHUB_DOMAIN=senhub.hide23.link
SENHUB_DB_URL=postgresql://senhub:senhubpass@127.0.0.1:5432/senhub
EOF

# ポート443を開放
ufw allow 443

# サービス再起動
systemctl restart senhub
```

## トラブルシューティング

### サービスが起動しない

```bash
journalctl -u senhub -n 50
```

よくある原因：
- `.env` の `SENHUB_DB_URL` が間違っている
- PostgreSQL が停止している → `service postgresql start`
- 証明書ファイルが見つからない → `SENHUB_USE_TLS=false` で一旦 HTTP 起動して確認

### Let's Encrypt 証明書取得に失敗する

- ポート80 がルーターで転送されているか確認
- DNS が正しいグローバル IP を向いているか確認
  ```bash
  python3 -c "import socket; print(socket.gethostbyname('senhub.hide23.link'))"
  ```

### DB 接続エラー

```bash
# PostgreSQL の状態確認
service postgresql status

# senhub ユーザーで接続テスト
PGPASSWORD=senhubpass psql -U senhub -d senhub -h 127.0.0.1 -c "\dt"
```
