## 構成A: Docker Compose（ローカルサーバー）

**実績環境:** Ubuntu 24.04 LTS on Proxmox (192.168.x.x)

### クイックインストール（推奨）

**手順はたった2ステップです。**

#### ① 設定ファイルを編集（初回のみ）

```bash
# 開発機で設定ファイルをコピーして編集
cp scripts/install.conf.example scripts/install.conf
```

`scripts/install.conf` を開き、サーバーの IP アドレスを記入します：

```bash
# scripts/install.conf
SERVER_DOMAIN="192.168.x.x"   # ← サーバーのIPアドレスに書き換える
```

> **注意:** `install.conf` は `.gitignore` で除外されており、Git には含まれません（IP アドレス保護）。

#### ② サーバーへ転送して実行

```bash
# 開発機から1コマンドで完了
rsync -av --exclude='__pycache__' --exclude='*.pyc' --exclude='*.egg-info' \
  /workspaces/example/ root@192.168.x.x:/tmp/senhub-src/ \
  && ssh root@192.168.x.x "bash /tmp/senhub-src/scripts/install-local.sh"
```

スクリプトが以下をすべて自動実行します：
- Docker CE + Compose インストール
- `/opt/senhub/` へのファイル配置
- Docker Compose 起動（TimescaleDB + Grafana）
- DBスキーマ適用 + Python 仮想環境セットアップ
- **channels.yaml をランダムキーで自動生成**（テストキー不要）
- systemd サービス登録・起動
- ヘルスチェック＆接続先 URL・チャンネルキー表示

完了後に表示される `writeKey` / `readKey` を Arduino / Python クライアントに設定してください。  
キーは `/opt/senhub/CREDENTIALS.txt`（chmod 600）にも保存されます。

> **冪等:** 2回目以降の実行も安全（既存の `.env` / `channels.yaml` は上書きしない）

### 手動インストール（ステップごとに確認したい場合）

### A-1. 前提パッケージのインストール

```bash
apt-get update
apt-get install -y git python3-pip python3-venv

# Docker インストール
apt-get install -y ca-certificates curl gnupg
install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | gpg --dearmor -o /etc/apt/keyrings/docker.gpg
chmod a+r /etc/apt/keyrings/docker.gpg
echo "deb [arch=amd64 signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu noble stable" \
  > /etc/apt/sources.list.d/docker.list
apt-get update
apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
```

### A-2. サーバーファイルの配置

```bash
mkdir -p /opt/senhub/server /opt/senhub/scripts
mkdir -p /opt/senhub/grafana/provisioning/datasources
mkdir -p /opt/senhub/grafana/provisioning/dashboards

# ファイルを scp または git で転送
# scp の場合（開発機から実行）:
scp server/main.py server/config.py server/db.py server/requirements.txt \
    server/schema.sql server/channels.yaml.example \
    root@192.168.x.x:/opt/senhub/server/
scp scripts/monitor.py scripts/gen-channel-keys.py \
    root@192.168.x.x:/opt/senhub/scripts/
```

### A-3. Python 仮想環境のセットアップ

```bash
cd /opt/senhub
python3 -m venv venv
source venv/bin/activate
pip install -r server/requirements.txt
```

### A-4. Docker Compose ファイルの作成

`/opt/senhub/docker-compose.yml` を作成：

```yaml
networks:
  senhub-net:
    name: senhub-net

volumes:
  timescaledb-data:
  grafana-data:

services:
  timescaledb:
    image: timescale/timescaledb:latest-pg16
    container_name: senhub-timescaledb
    restart: unless-stopped
    networks:
      - senhub-net
    ports:
      - "5432:5432"
    environment:
      POSTGRES_USER: senhub
      POSTGRES_PASSWORD: senhubpass
      POSTGRES_DB: senhub
    volumes:
      - timescaledb-data:/var/lib/postgresql/data

  grafana:
    image: grafana/grafana:latest
    container_name: senhub-grafana
    restart: unless-stopped
    networks:
      - senhub-net
    ports:
      - "3000:3000"
    environment:
      GF_SECURITY_ADMIN_USER: admin
      GF_SECURITY_ADMIN_PASSWORD: senhub123
      GF_USERS_ALLOW_SIGN_UP: "false"
    volumes:
      - grafana-data:/var/lib/grafana
      - /opt/senhub/grafana/provisioning:/etc/grafana/provisioning
```

> **ポイント:** コンテナ間は `senhub-net` の名前付きネットワークで接続するため、
> IPアドレスに依存せず `senhub-timescaledb:5432` でアクセスできる（再起動後も変わらない）。

### A-5. Grafana データソース自動プロビジョニング

`/opt/senhub/grafana/provisioning/datasources/timescaledb.yaml` を作成：

```yaml
apiVersion: 1

datasources:
  - name: TimescaleDB
    type: grafana-postgresql-datasource
    access: proxy
    url: senhub-timescaledb:5432
    database: senhub
    user: senhub
    secureJsonData:
      password: senhubpass
    jsonData:
      sslmode: disable
      postgresVersion: 1600
      timescaledb: true
    isDefault: true
    editable: true
```

> このファイルがあれば Grafana 起動時に自動でデータソースが設定される（手動設定不要）。

### A-6. Docker Compose 起動 & スキーマ適用

```bash
cd /opt/senhub
docker compose up -d

# DB の準備完了を待つ（約10秒）
sleep 10
docker exec -i senhub-timescaledb psql -U senhub -d senhub < server/schema.sql
```

### A-7. channels.yaml の作成

```bash
cp /opt/senhub/server/channels.yaml.example /opt/senhub/server/channels.yaml
chmod 600 /opt/senhub/server/channels.yaml
# エディタで write_key / read_key を設定（→ 共通セクション参照）
```

### A-8. .env の作成

```bash
cat > /opt/senhub/server/.env << 'EOF'
SENHUB_PORT=8000
SENHUB_USE_TLS=false
SENHUB_DB_URL=postgresql://senhub:senhubpass@localhost:5432/senhub
EOF
```

### A-9. systemd サービス登録

```bash
cat > /etc/systemd/system/senhub.service << 'EOF'
[Unit]
Description=Senhub IoT Data Server
After=network.target docker.service
Requires=docker.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/senhub/server
EnvironmentFile=/opt/senhub/server/.env
ExecStart=/opt/senhub/venv/bin/python main.py
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable senhub
systemctl start senhub
```

### A-10. 動作確認

```bash
# readKey を CREDENTIALS.txt から取得
READ_KEY=$(grep CH_READ_KEY /root/senhub-CREDENTIALS.txt | cut -d'"' -f2)

# Senhub API テスト
curl "http://localhost:8000/api/v1/channels/100/data?readKey=${READ_KEY}"

# Grafana（ブラウザ）
# http://192.168.x.x:3000
# ユーザー: admin / パスワード: CREDENTIALS.txt の GRAFANA_PASS を参照
cat /root/senhub-CREDENTIALS.txt
```

### A-11. リアルタイムモニター

```bash
READ_KEY=$(grep CH_READ_KEY /root/senhub-CREDENTIALS.txt | cut -d'"' -f2)
python3 /opt/senhub/scripts/monitor.py \
  --url http://localhost:8000/api/v1 \
  --key "${READ_KEY}" \
  --interval 2
```

## 構成B: ベアメタル + Let's Encrypt（インターネット公開）

**実績環境:** Ubuntu 24.04 LTS (senhub.example.com / 192.168.x.x)

### クイックインストール（推奨）

**手順はたった2ステップです。**

#### ① 設定ファイルを編集（初回のみ）

```bash
cp scripts/install.conf.example scripts/install.conf
```

`scripts/install.conf` を開いて設定します：

```bash
# HTTPS 本番の場合（Let's Encrypt で証明書を自動取得）
SERVER_DOMAIN="senhub.example.com"   # ← 自分のドメイン名
USE_TLS="true"
LE_EMAIL="admin@example.com"          # ← Let's Encrypt 登録メール

# HTTP のみの場合（社内 LAN / テスト環境）
# SERVER_DOMAIN="192.168.1.100"
# USE_TLS="false"
```

> **注意:** `install.conf` は `.gitignore` で除外されており、Git には含まれません。

#### ② サーバーへ転送して実行

```bash
# 開発機から1コマンドで完了
rsync -av --exclude='__pycache__' --exclude='*.pyc' --exclude='*.egg-info' \
  /workspaces/example/ root@SERVER_IP:/tmp/senhub-src/ \
  && ssh root@SERVER_IP "bash /tmp/senhub-src/scripts/install-production.sh"
```

CLI 引数でも設定可能（引数 > install.conf の優先順位）：

| オプション | 説明 | install.conf の対応キー |
|-----------|------|------------------------|
| `--domain DOMAIN` | ドメイン名または IP | `SERVER_DOMAIN` |
| `--email EMAIL` | Let's Encrypt 用メール | `LE_EMAIL` |
| `--no-tls` | HTTPS をスキップ | `USE_TLS=false` |
| `--skip-db` | DB 初期化をスキップ | （CLI 引数のみ）|
| `--port PORT` | リッスンポート | `SERVER_PORT` |

### 手動インストール（ステップごとに確認したい場合）

### B-1. 前提条件

- ドメイン名のDNS AレコードがサーバーのグローバルIPに向いていること
- ルーターでポート80・443をサーバーに転送していること

### B-2. TimescaleDB + PostgreSQL インストール

```bash
apt-get update
apt-get install -y gnupg curl ca-certificates

# TimescaleDB リポジトリの登録
curl -fsSL https://packagecloud.io/timescale/timescaledb/gpgkey \
  | gpg --dearmor -o /usr/share/keyrings/timescaledb.gpg
echo 'deb [signed-by=/usr/share/keyrings/timescaledb.gpg] https://packagecloud.io/timescale/timescaledb/ubuntu/ noble main' \
  > /etc/apt/sources.list.d/timescaledb.list

apt-get update
apt-get install -y timescaledb-2-postgresql-16 postgresql-client-16
```

### B-3. PostgreSQL / DB セットアップ

```bash
# TimescaleDB を preload に追加
sed -i "s/#shared_preload_libraries = ''/shared_preload_libraries = 'timescaledb'/" \
  /etc/postgresql/16/main/postgresql.conf
sed -i 's/^local   all             postgres                                peer/local   all             postgres                                trust/' \
  /etc/postgresql/16/main/pg_hba.conf

service postgresql start && service postgresql reload

# senhub ユーザー・DB 作成
psql -U postgres -c "CREATE USER senhub WITH PASSWORD 'senhubpass';"
psql -U postgres -c "CREATE DATABASE senhub OWNER senhub;"
psql -U postgres -d senhub -c "CREATE EXTENSION IF NOT EXISTS timescaledb CASCADE;"

# pg_hba.conf を peer 認証に戻す
sed -i 's/^local   all             postgres                                trust/local   all             postgres                                peer/' \
  /etc/postgresql/16/main/pg_hba.conf
service postgresql reload
```

### B-4. スキーマ適用

```bash
PGPASSWORD=senhubpass psql -U senhub -d senhub -h 127.0.0.1 -f server/schema.sql
```

作成されるテーブル・ビュー：

| 名前 | 種別 | 内容 |
|------|------|------|
| `channels` | テーブル | チャンネル認証キー管理 |
| `channel_properties` | テーブル | フィールドのメタ情報（name/unit/type） |
| `sensor_data` | Hypertable | 連続センサーデータ |
| `events` | Hypertable | ON/OFF イベントログ |
| `sensor_data_1min` | 集約ビュー | 1分平均（Continuous Aggregate） |
| `sensor_data_1hour` | 集約ビュー | 1時間平均（Continuous Aggregate） |

### B-5. サーバーファイルのデプロイ

```bash
mkdir -p /opt/senhub/server /opt/senhub/scripts
cp server/main.py server/db.py server/config.py server/requirements.txt \

cp scripts/monitor.py scripts/gen-channel-keys.py /opt/senhub/scripts/
```

### B-9. HTTPS 対応（Let's Encrypt）

```bash
apt-get install -y certbot
systemctl stop senhub
ufw allow 80

certbot certonly --standalone --non-interactive --agree-tos \
  --email your-email@example.com \
  -d senhub.example.com

# .env を HTTPS 設定に更新
cat > /opt/senhub/server/.env << 'EOF'
SENHUB_USE_TLS=true
SENHUB_PORT=443
SENHUB_DOMAIN=senhub.example.com
SENHUB_DB_URL=postgresql://senhub:senhubpass@127.0.0.1:5432/senhub
EOF

ufw allow 443
systemctl restart senhub
```

> 証明書は certbot.timer により **90日ごとに自動更新**されます。

### B-10. ファイアウォール設定

```bash
ufw allow 443   # HTTPS（本番）
ufw allow 80    # 証明書取得時のみ
ufw allow 8000  # HTTP（社内LAN・開発時）
```

## トラブルシューティング

### サービスが起動しない

```bash
journalctl -u senhub -n 50
```

よくある原因：
- `.env` の `SENHUB_DB_URL` が間違っている
- PostgreSQL / Docker が停止している
- 証明書ファイルが見つからない → `SENHUB_USE_TLS=false` で HTTP 起動して確認

### Grafana が DB に接続できない（構成A）

- 再起動後に Grafana コンテナの IP が変わることがある
- **対策済み**: `docker-compose.yml` で名前付きネットワーク `senhub-net` を使用し、
  `senhub-timescaledb:5432` というコンテナ名で接続するため IP 依存なし
- `grafana/provisioning/datasources/timescaledb.yaml` に URL が記述されており、
  Grafana 再起動時に自動で設定される

### タイムスタンプが UTC になる

時刻が日本時間（JST = UTC+9）ではなく UTC で表示される場合：

```bash
# サーバーのタイムゾーン確認
timedatectl

# JST に設定
timedatectl set-timezone Asia/Tokyo

# senhub サービス再起動
systemctl restart senhub
```

> `/opt/senhub/server/.env` に `TZ=Asia/Tokyo` が含まれていることも確認してください。  
> TimescaleDB / Grafana コンテナも `TZ: Asia/Tokyo` が設定されています（`docker-compose.yml`）。

### タイムスタンプが 1970年になる（NTP 未同期デバイス）

- サーバー側で 2020年以前のタイムスタンプは自動的に受信時刻で上書きする
- Arduino 側は `configTime()` で NTP 同期を行うことを推奨（`SenhubDebug.ino` 参照）

### Let's Encrypt 証明書取得に失敗する（構成B）

- ポート80がルーターで転送されているか確認
- DNS が正しいグローバル IP を向いているか確認

```bash
python3 -c "import socket; print(socket.gethostbyname('senhub.example.com'))"
```
