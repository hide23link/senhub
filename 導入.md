# Senhub 導入ガイド（初心者向け）

ESP32 のセンサーデータを収集・グラフ化する IoT システムです。  
このガイドは **社内 LAN（HTTP）** 構成を前提にしています。

---

## 全体の流れ

```
【Ubuntu サーバー】                     【ESP32】

① IP アドレスを確認
② GitHub からソースを取得
③ install.conf を編集
④ インストールスクリプト実行
   → API サーバー起動
   → Grafana 起動
   → writeKey / readKey 発行
                                        ⑤ Arduino に writeKey を書いて書き込み
                                        ⑥ Grafana でデータを確認  ✅
```

---

## 必要なもの

| 種類 | 内容 |
|------|------|
| **サーバー機** | Ubuntu 24.04 LTS（RAM 2GB 以上）|
| **ESP32** | Wi-Fi 内蔵の Arduino 互換マイコン |
| **ネットワーク** | サーバーと ESP32 が同じ LAN にいること |

---

## ① サーバーの IP アドレスを確認する

サーバーのターミナルで実行します。

```bash
ip a | grep "inet "
```

出力例：
```
inet 127.0.0.1/8 ...
inet 192.168.1.100/24 ...   ← これがサーバーの IP アドレス
```

> 以降のコマンドで `192.168.1.100` と書かれている部分は、  
> ここで確認した実際の IP アドレスに置き換えてください。

---

## ② GitHub からソースを取得する

```bash
# git をインストール
apt update && apt install -y git

# リポジトリをクローン
git clone https://github.com/hide23link/senhub.git

# senhub フォルダに移動
cd senhub
```

---

## ③ install.conf を作成する

`192.168.1.100` の部分を **① で確認した IP アドレス** に変えて実行します。

```bash
echo 'SERVER_DOMAIN="192.168.1.100"' > scripts/install.conf
```

設定を確認：

```bash
cat scripts/install.conf
# → SERVER_DOMAIN="192.168.1.100" と表示されれば OK
```

---

## ④ インストールスクリプトを実行する

```bash
bash scripts/install-local.sh
```

完了まで **約3〜5分** かかります。  
以下がすべて自動でセットアップされます：

- Docker（コンテナ実行環境）
- TimescaleDB（時系列データベース）
- Grafana（グラフ表示ツール）
- Senhub API サーバー

完了すると以下のような画面が表示されます：

```
========================================
  Senhub インストール完了
========================================
  API      : http://192.168.1.100:8000
  Grafana  : http://192.168.1.100:3000
  チャンネル ID : 100
  writeKey : w_xxxxxxxxxxxxxxxxxxxxxxxx
  readKey  : r_xxxxxxxxxxxxxxxxxxxxxxxx
  Grafana  : admin / xxxxxxxxxxxxxxxx
========================================
キーは /root/senhub-CREDENTIALS.txt に保存済み
```

> ⚠️ **`writeKey` と `readKey` と Grafana パスワードをメモしてください。**  
> 後から確認したい場合は `cat /root/senhub-CREDENTIALS.txt` で確認できます。

## ⑤ ESP32 に書き込む

### Arduino ライブラリのインストール

1. [GitHub Releases](https://github.com/hide23link/senhub/releases) から `Senhub-x.x.x.zip` をダウンロード
2. Arduino IDE を開く
3. メニュー：**スケッチ → ライブラリをインクルード → .ZIP形式のライブラリをインストール**
4. ダウンロードした ZIP を選択

### サンプルスケッチを開く

メニュー：**ファイル → スケッチ例 → Senhub → SenhubTest**

### 設定を書き換える

スケッチの冒頭にある以下の部分だけ変更します：

```cpp
// ===== ここだけ変更する =====
const char* WIFI_SSID  = "あなたのWi-Fiの名前";      // ← Wi-Fi SSID
const char* WIFI_PASS  = "あなたのWi-Fiのパスワード"; // ← Wi-Fi パスワード
const char* WRITE_KEY  = "w_xxxxxxxx...";             // ← ④ の writeKey

const char* SERVER_URL = "http://192.168.1.100:8000/api/v1"; // ← ① の IP アドレス
// ============================
```

> `#define USE_TLS false` は社内 LAN（HTTP）なので **変更不要** です。

### ESP32 に書き込む

1. ESP32 を USB で PC に接続
2. Arduino IDE でボードとポートを選択
3. **書き込みボタン（→）** をクリック

---

## ⑥ Grafana でデータを確認する

ブラウザで **`http://192.168.1.100:3000`** を開きます。  
（IP アドレスは ① で確認したもの）

| 項目 | 値 |
|------|-----|
| ユーザー名 | `admin` |
| パスワード | ④ に表示された Grafana パスワード |

ログイン後、左メニューの **Explore** からデータを確認できます。

---

## うまくいかない時

### ESP32 がデータを送れない

Arduino IDE のシリアルモニタ（115200bps）を開いて確認します。

| 表示 | 意味 |
|------|------|
| `HTTP=200 OK` | 正常に送信できている |
| `HTTP=401` | writeKey が間違っている |
| `HTTP=-1` | サーバーに接続できない（IP アドレスや Wi-Fi を確認）|

### サーバーが応答しない

```bash
# サーバー上で確認
systemctl status senhub           # API サーバーの状態
docker compose -f /opt/senhub/docker-compose.yml ps   # DB の状態

# API の疎通確認（サーバー上で）
curl http://localhost:8000/health
# → {"status":"ok"} が返れば OK
```

### インストールをやり直したい

```bash
# サーバー上で実行
docker compose -f /opt/senhub/docker-compose.yml down -v
rm -rf /opt/senhub
cd ~/senhub
bash scripts/install-local.sh
```
